Create nginx/nginx.prod.conf for production HTTPS with the following configuration (per D-05, D-07, D-08, D-17):

1. **map directive for WebSocket upgrade** (copy from existing nginx.conf):
```nginx
map $http_upgrade $connection_upgrade {
    default upgrade;
    ''      close;
}
```

2. **upstream backend block** (copy from existing):
```nginx
upstream backend {
    server backend:8000;
}
```

3. **HTTP server block** (port 80) - ACME challenge location and HTTPS redirect:
```nginx
server {
    listen 80;
    server_name _;  # Will be replaced with actual domain

    # ACME challenge for Let's Encrypt certificate renewal
    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }

    # Redirect all other traffic to HTTPS
    location / {
        return 301 https://$host$request_uri;
    }
}
```

4. **HTTPS server block** (port 443) with SSL and WebSocket proxy:
```nginx
server {
    listen 443 ssl http2;
    server_name _;  # Will be replaced with actual domain

    # SSL certificates from Let's Encrypt (mounted via docker-compose.prod.yml)
    ssl_certificate /etc/letsencrypt/live/DOMAIN/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/DOMAIN/privkey.pem;

    # Modern SSL configuration (per RESEARCH.md)
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers off;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 1d;

    # Frontend static files
    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
    }

    # Socket.io WebSocket endpoint (preserving 24-hour timeouts from Phase 5)
    location /socket.io/ {
        proxy_pass http://backend;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection $connection_upgrade;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 86400s;
        proxy_send_timeout 86400s;
        proxy_buffering off;
    }

    # Backend API proxy
    location /api/ {
        proxy_pass http://backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # Health check endpoint (no auth required - per RESEARCH.md recommendation)
    location /health {
        proxy_pass http://backend;
        proxy_set_header Host $host;
    }
}
```

Key requirements:
- Use `map $http_upgrade $connection_upgrade` for WebSocket (same pattern as existing nginx.conf)
- Preserve `proxy_read_timeout 86400s` for 24-hour WebSocket connections (SEC-03)
- Use placeholder `DOMAIN` for ssl_certificate paths (actual domain configured on server)
- Include ACME challenge location for certificate renewal (D-05, D-06)
- HTTP server returns 301 redirect to HTTPS (D-07)
- Health check has no auth (for Docker health checks)

Create systemd service and timer for automatic certificate renewal (per D-06).

**deploy/certbot-renew.service:**
```ini
[Unit]
Description=Let's Encrypt certificate renewal for The Holding Cell
After=network-online.target
Wants=network-online.target

[Service]
Type=oneshot
# Stop nginx to free port 80, renew certificates, start nginx
ExecStart=/usr/bin/docker compose -f /opt/holding-cell/docker-compose.yml -f /opt/holding-cell/docker-compose.prod.yml stop nginx
ExecStart=/usr/bin/certbot renew --quiet --standalone
ExecStart=/usr/bin/docker compose -f /opt/holding-cell/docker-compose.yml -f /opt/holding-cell/docker-compose.prod.yml start nginx

[Install]
WantedBy=multi-user.target
```

**deploy/certbot-renew.timer:**
```ini
[Unit]
Description=Twice daily renewal of Let's Encrypt certificates

[Timer]
OnCalendar=*-*-* 00,12:00:00
RandomizedDelaySec=3600
Persistent=true

[Install]
WantedBy=timers.target
```

Key requirements:
- Service stops nginx before renewal (port 80 needed for standalone mode)
- Uses standalone mode (--standalone) per D-05
- Timer runs twice daily at 00:00 and 12:00 (per D-06)
- RandomizedDelaySec prevents thundering herd if multiple servers renew simultaneously
- Persistent=true ensures renewal happens after server downtime
- Path /opt/holding-cell is placeholder - actual path configured during deployment
