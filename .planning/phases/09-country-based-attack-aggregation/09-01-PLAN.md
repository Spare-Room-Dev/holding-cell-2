    Create `frontend/src/hooks/useCountryPrisoners.ts` implementing the `useCountryPrisoners` hook.

    The hook accepts an `attacks` array (AttackEvent[]) and returns a `CountryPrisoner[]`.

    Implementation details:
    1. Import `useMemo` and `useRef` from React. Import `AttackEvent`, `Archetype`, `CountryPrisoner` from `@/types/attack`. Import `getCountryName` from `@/utils/countryNames`.

    2. Create the hook function `useCountryPrisoners(attacks: AttackEvent[]): CountryPrisoner[]`.

    3. Use `useRef<Set<string>>` (initialized to `new Set()`) to track previously seen country codes for `isNew` detection. Per Pitfall 1 from RESEARCH.md: compare the new set of country codes against the previous set. Countries not in the previous set get `isNew: true`.

    4. Use `useRef<Record<string, number>>` (initialized to `{}`) to track previous counts per country code for `isUpdated` detection. Per Pitfall 4: a country is `isUpdated` only when `prevCount > 0 && currentCount > prevCount`. This prevents the pulse from triggering on first appearance (which uses the entrance spring instead) and on initial page load/history batch.

    5. Inside `useMemo` (dependency: `[attacks]`):
       - Create a `Map<string, { countryCode, countryName, count, archetypes, lastAttack }>` to aggregate.
       - Iterate through `attacks` array. For each attack:
         - If country code exists in map: increment count, increment archetype count, update lastAttack if newer.
         - If not: create new entry with countryCode, countryName (from getCountryName), count: 1, archetypes: { [attack.archetype]: 1 }, lastAttack: attack.timestamp.
       - Convert map to array. For each entry, compute `isNew` (countryCode not in prevCountrySetRef) and `isUpdated` (prevCountsRef has a count > 0 for this country AND current count > previous count).
       - Sort by count descending.
       - Apply 20-country cap: `slice(0, 20)` per D-03.
       - After computing, update `prevCountrySetRef.current` with the full set of country codes from this batch (all countries before slicing, so we remember all seen countries).
       - Update `prevCountsRef.current` with the current count for each country in the capped array.
       - Return the capped array.

    6. Export the hook as a named export.

    Important implementation notes:
    - The `isNew` flag is `true` only on the render where a country first appears. On subsequent renders, `isNew` is `false` because the country is now in the previous set.
    - The `isUpdated` flag is `true` only when a country's count increases AND it had a previous count > 0. On initial appearance, `isUpdated` is `false` (first appearance uses entrance spring, not pulse).
    - When `attack_history` arrives on reconnect (batch load), all countries have `prevCount === 0` so `isUpdated` is `false` for all of them. This prevents the "all prisoners flash on page load" bug described in Pitfall 4.
    - The sorting and capping happen AFTER computing isNew/isUpdated, so countries that fall out of the top 20 are properly tracked in prevCountsRef for when they re-enter.
  </action>
