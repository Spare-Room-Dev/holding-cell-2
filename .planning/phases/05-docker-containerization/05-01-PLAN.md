  <action>
    Create directory `nginx/` and file `nginx/nginx.conf` with WebSocket proxy configuration.

    **nginx.conf structure (per RESEARCH.md Pattern 3):**

    ```nginx
    # Upstream for backend service (Docker Compose service name)
    upstream backend {
        server backend:8000;
    }

    server {
        listen 80;
        server_name localhost;

        # Frontend static files (served from nginx container)
        location / {
            root /usr/share/nginx/html;
            try_files $uri $uri/ /index.html;
        }

        # Socket.io WebSocket endpoint (per D-02)
        location /socket.io/ {
            proxy_pass http://backend;
            proxy_http_version 1.1;

            # Required for WebSocket upgrade (prevents 60-second disconnects)
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";

            # Standard proxy headers
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

            # Extended timeouts for long-lived WebSocket connections
            proxy_read_timeout 86400s;
            proxy_send_timeout 86400s;
        }

        # Backend API proxy (for future health check routing)
        location /api/ {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }

        # Health check endpoint passthrough
        location /health {
            proxy_pass http://backend;
            proxy_set_header Host $host;
        }
    }
    ```

    **Critical WebSocket headers (per SEC-03):**
    - `proxy_http_version 1.1` - Required for WebSocket upgrade
    - `proxy_set_header Upgrade $http_upgrade` - Passes upgrade header
    - `proxy_set_header Connection "upgrade"` - Signals WebSocket upgrade
    - `proxy_read_timeout 86400s` - 24-hour timeout prevents 60-second disconnects

    The upstream `backend:8000` uses Docker Compose service name resolution (not localhost).
  </action>
