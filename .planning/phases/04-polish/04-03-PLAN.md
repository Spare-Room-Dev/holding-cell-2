## Step 1: Start both backend and frontend
```bash
npm run dev:all
```

## Step 2: Verify Responsive Layout
1. Open http://localhost:3000 in browser
2. Open browser DevTools (F12)
3. Toggle device toolbar for responsive testing
4. Test breakpoints:
   - **Desktop (>=1024px)**: Verify 70/30 sidebar layout shows JailCellGrid (left, 70%) and StatsPanel (right, 30%)
   - **Tablet (768px-1023px)**: Verify stacked layout (JailCellGrid top, stats area shows mobile button)
   - **Mobile (<768px)**: Verify full-width JailCellGrid, no visible sidebar, "Stats" button in header

5. On mobile view (<768px), click "Stats" button:
   - Verify bottom sheet slides up from bottom
   - Verify backdrop appears (semi-transparent black)
   - Click outside backdrop → sheet should close
   - Swipe down on sheet → sheet should close

## Step 3: Verify Light/Dark Mode Toggle
1. Find the theme toggle button (sun/moon icon) in the header next to "LIVE" status
2. Click the toggle:
   - Verify dark mode is default on page load
   - Click toggle → verify light mode activates (background changes to #F5F4F0, surfaces to white)
   - Click again → verify return to dark mode
   - Refresh page → verify dark mode is still default (no localStorage persistence)

## Step 4: Verify Archetype Classification
Watch the backend console output and the frontend prisoner display. Verify that generated attacks match BACK-08 rules:

| Archetype | Expected Behavior |
|-----------|-------------------|
| script_kiddie | Duration <2min, commands <10, basic commands like whoami, ls, cat |
| botnet_drone | Repeated password attempts (admin/admin, root/123456, etc.) |
| apt_operative | Duration >10min, commands >50, recon commands (ls -la, netstat -an, etc.) |
| iot_worm | Commands contain busybox, buildroot, or MIPS patterns |
| hacktivist | Username contains anonymous, free, hack, or similar patterns |

Observe 5-10 attack events in the console. Each should match one archetype pattern.

## Step 5: Verify Number Formatting
1. Let attacks accumulate (or refresh page to reset)
2. Check that counter numbers display with commas (e.g., "1,234" not "1234")
3. Verify cap display: Counters should show "99,999+" when exceeding 99,999

## Step 6: Verify Empty State (if needed)
1. Refresh the page quickly to catch empty state
2. Verify "The cell is empty. Waiting for attackers..." displays correctly
3. Verify "SIGNAL LOST" banner appears when backend is stopped

## Step 7: Verify Disconnect UX
1. Stop the backend (Ctrl+C in terminal)
2. Verify "SIGNAL LOST" banner appears on frontend
3. Restart backend
4. Verify "LIVE" badge returns with phosphor green glow
